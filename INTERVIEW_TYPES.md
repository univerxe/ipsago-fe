# 🎯 면접 유형 가이드 / Interview Types Guide

## 📋 3가지 면접 유형

### 1️⃣ 일반 면접 (Standard Interview)
**균형잡힌 종합 면접**

**특징:**
- 기술과 인성을 모두 평가
- 가장 일반적인 면접 형태
- 자기소개부터 시작

**질문 예시:**
- 1분 자기소개 부탁드립니다
- 이 직무에 지원하신 이유는 무엇인가요?
- 프로젝트 경험을 말씀해 주세요
- 팀워크 경험이 있나요?
- 장점과 단점은 무엇인가요?

**추천 대상:**
- 처음 면접 연습하는 사람
- 전반적인 면접 준비
- 실제 면접과 가장 유사한 경험

---

### 2️⃣ 인성 면접 (Personality Test)
**가치관과 태도 평가**

**특징:**
- 인성, 가치관, 행동 패턴 평가
- STAR 방식 답변 중요
- 실제 경험 기반 질문

**주요 질문 영역:**
1. **팀워크 & 협업**
   - 팀원과 의견 충돌이 있었던 경험
   - 다른 팀과 협업한 경험
   
2. **갈등 해결**
   - 어려운 상황을 극복한 경험
   - 스트레스 관리 방법

3. **리더십**
   - 팀을 이끈 경험
   - 후배/동료를 도운 경험

4. **실패와 성장**
   - 실패한 프로젝트 경험
   - 실수로부터 배운 점

5. **윤리와 가치관**
   - 윤리적 딜레마 상황
   - 회사 문화 적응

**답변 팁:**
- STAR 방식 사용 (Situation, Task, Action, Result)
- 구체적인 사례 제시
- 배운 점과 성장 강조
- 긍정적인 태도 유지

**예시 질문:**
```
Q: 팀원과 의견이 달랐던 경험과 어떻게 해결했는지 말씀해 주세요.
Q: 실패한 프로젝트 경험과 그로부터 배운 점은 무엇인가요?
Q: 스트레스를 받는 상황에서 어떻게 대처하시나요?
Q: 5년 후 커리어 목표는 무엇인가요?
```

---

### 3️⃣ 기술 면접 (Technical Interview)
**실무 능력 심층 평가**

**특징:**
- 기술 역량 집중 평가
- 구체적인 기술 질문
- 문제 해결 과정 중요

**주요 질문 영역:**

**개발자:**
1. **기술 스택 경험**
   - React, TypeScript, Node.js 등
   - 프레임워크 선택 이유
   
2. **프로젝트 아키텍처**
   - 시스템 설계 방법
   - 데이터베이스 설계
   
3. **코드 품질**
   - 클린 코드 원칙
   - 리팩토링 경험
   
4. **성능 최적화**
   - 병목 지점 파악 방법
   - 최적화 경험과 결과
   
5. **문제 해결**
   - 버그 해결 과정
   - 트러블슈팅 경험

**디자이너:**
1. **디자인 프로세스**
2. **툴 사용 능력** (Figma, Adobe XD)
3. **사용자 리서치**
4. **디자인 시스템**
5. **A/B 테스팅**

**비즈니스 분석가:**
1. **데이터 분석 도구** (SQL, Python, Excel)
2. **통계 지식**
3. **리포트 작성**
4. **비즈니스 인사이트 도출**
5. **의사결정 지원**

**답변 팁:**
- 구체적인 숫자와 데이터 제시
- Before & After 비교
- 기술 선택의 이유 설명
- Trade-off 이해도 표현
- 최신 기술 트렌드 언급

**예시 질문:**
```
Q: React에서 성능 최적화를 어떻게 하셨나요?
Q: 왜 TypeScript를 사용하시나요? JavaScript와의 차이는?
Q: 데이터베이스 인덱스를 어떻게 설계하셨나요?
Q: 대용량 트래픽을 어떻게 처리하셨나요?
Q: CI/CD 파이프라인 구축 경험이 있나요?
```

## 🎯 면접 유형별 준비 전략

### 일반 면접 준비:
1. ✅ 자기소개 (1분) 완벽하게 준비
2. ✅ 지원 동기 명확히
3. ✅ 주요 프로젝트 3개 정리
4. ✅ 강점/약점 파악
5. ✅ 회사 리서치

### 인성 면접 준비:
1. ✅ STAR 방식 사례 5개 이상
2. ✅ 팀워크 경험 정리
3. ✅ 갈등 해결 사례
4. ✅ 실패 경험과 교훈
5. ✅ 커리어 목표 구체화

### 기술 면접 준비:
1. ✅ 이력서의 모든 기술 설명 가능하게
2. ✅ 프로젝트 아키텍처 다이어그램 준비
3. ✅ 문제 해결 사례 (버그, 최적화)
4. ✅ 최신 기술 트렌드 학습
5. ✅ 코드 리뷰 경험

## 💡 면접 유형 전환 방법

### 인터페이스 사용법:
```
1. 면접 시작 시 상단 네비게이션 바 확인
2. 세 가지 버튼 중 원하는 면접 유형 선택:
   - 일반 면접
   - 인성 면접
   - 기술 면접
3. 버튼 클릭 시 AI가 해당 유형에 맞는 질문 시작
4. 언제든 유형 변경 가능
```

## 🎓 효과적인 연습 방법

### 1단계: 일반 면접으로 시작
- 전반적인 흐름 파악
- 기본 답변 준비

### 2단계: 약한 부분 집중
- 기술이 약하다면 → 기술 면접
- 경험 사례가 부족하다면 → 인성 면접

### 3단계: 실전 연습
- 일반 면접으로 종합 점검
- 실제 면접처럼 진지하게

### 4단계: 반복
- 여러 번 연습
- 답변 개선
- 자신감 향상
